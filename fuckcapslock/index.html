fuckcapslock.js
Fuckcapslock is a JavaScript library that negotiates effect of the CapsLock button — case of typed characters is not affected any longer, and the whole page behaves as if CapsLock was never active.
Why?

The CapsLock button has extremely attractive position: it is located on the home row — the one in the middle, where your hands lie relaxed. It is therefore easily accessible without bending the fingers. In fact, CapsLock is accessible easier than any other non-letter key (perhaps with except for Spacebar).

For historical reasons, the most accessible button has the most stupid function assigned to it — TO MAKE ALL THE TYPED CHARACTERS APPEAR IN CAPS. This rudimental side-effect made the button useless for other purposes as a modifier.

Until now. With fuckcapslock.js the state of the CapsLock key is disregarded. There is yet an opprotunity to handle its events and reuse the button as a convenient hotkey. It exposes the keyboard events in the normal way and can be identified by the key code 20 or code 'CapsLock'. Just like as this indicator reflects you pushing the CapsLock:

There is no more need to warn user that his CapsLock is on before entering passwords or any other data, if he joined the session with CapsLock active by mistake. It can be ignored, simply and transparently.

Also you can inculcate your users the good manners so that they type in a normal case.

Acutally we can live without it.

Text in all capitals is harder to read (it has all the same line shape). If you still wish to use the CapsLock when typing an expressive message implying to SHOUT OUT LOUD, just remind yourself that in real life shouting requires some extra effort, therefore you might consider to make a little effort when typing as well by HOLDING THE FUCKING SHIFT.

Note on frames: fuckcapslock.js tries to burrow recursively into the subframes (even if those are created in the runtime dynamically), but it will only initialize within directly accessible ones — the frames created within the same application. Cross-origin frames are protected by browser against external fuckapslock.js initialization. The solution is to explicitly load fuckapslock.js into those frames as well.

Of course the CapsLock button state will only be properly disregarded within the browser tab where fuckcapslock.js is included, and if user leaves the tab having the CapsLock on, it will distract (but we have to start somewhere).

What exactly fuckcapslock.js does

It tracks the state of the CapsLock upon keyboard events, and if the CapsLock is on, it does the following:

  • Changing the content of the inputs, textareas and content-editable divs is prevented, the library then inserts the characters that would appear if the CapsLock was inactive;
  • Keyboard events (keyup, keydown and keypress) are taken over, suppressed, and re-emitted with updated properties as those would be if the CapsLock was inactive.
